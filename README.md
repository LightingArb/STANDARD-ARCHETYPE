# STANDARD-ARCHETYPE

## 這是什麼？

一套讓 AI 幫你管專案的系統。你說一句話，AI 幫你規劃、討論、整理、甚至自己推進整個專案。

它包含兩個部分：

- **專案管理** — 結構化的專案流程，從開案到施工都有規則和閘門，確保 AI 不會亂跑
- **熔爐百相** — AI 人格與角色的供應庫，讓 AI 在討論時能切換不同專業視角（產品策略、工程施工、UI 設計、資安審計⋯⋯）

兩者搭配使用：專案管理負責流程和規則，熔爐百相負責讓 AI 有腦子。

---

## 能幫我做什麼？

| 你想做的事 | 你說 | AI 做什麼 |
|------------|------|-----------|
| 開始一個新專案 | `開一般專案 記帳App` | 建好資料夾結構，開始幫你規劃 |
| 接續推進手上的專案 | `討論專案` | 自動判斷進度，從斷點接著做 |
| 多角度腦力激盪 | `發散思考 定價策略` | 召集不同角色的 AI 一起討論 |
| 讓 AI 自己跑一段 | `啟動 Harness` | AI 自主推進，遇到大事停下來問你 |
| 整理混亂的資料 | `整理 參考資料` | 掃描、分類、歸位 |

完整指令清單在 [`專案管理/指令.md`](專案管理/指令.md)。

## 怎麼開始

### 1. 複製這個 repo

```bash
git clone <repo-url>
```

### 2. 看指令手冊

打開 [`專案管理/指令.md`](專案管理/指令.md)，上半段有場景導覽，找到你想做的事就行。

### 3. 建你的第一個專案

跟 AI 說：

```
開一般專案 我的第一個專案
```

AI 會幫你建好結構，然後你用 `討論專案` 開始推進。

### 4.（進階）讓 AI 自己跑

當專案有了基本方向，想讓 AI 自主推進：

```
啟動 Harness
```

詳細說明在 [`專案管理/大絕招.md`](專案管理/大絕招.md)。

## 三種操作模式

這套系統有三種「態」，你可以根據當下需要隨時切換：

| 模式 | 白話說明 | 入口文件 |
|------|----------|----------|
| **手動模式** | 你自己照結構操作，AI 只是工具 | [`專案管理使用手冊.md`](專案管理/_系統/基地/專案管理使用手冊.md) |
| **指令模式** | 你說一句話，AI 幫你做 | [`指令.md`](專案管理/指令.md) |
| **自主模式（Harness）** | AI 自己跑，你只管重大決定 | [`大絕招.md`](專案管理/大絕招.md) |

大多數人日常用**指令模式**就夠了。

## 熔爐百相是什麼？

熔爐百相是 AI 角色的素材庫。它從各種開源專案和技能文件中提煉出結構化的「人格卡」和「角色卡」，讓 AI 在專案討論時能扮演不同專業角色。

你不需要手動管理它。系統會在需要時自動從熔爐百相取用合適的角色。如果你想手動指定，可以用 `取用人格` 指令。

內容包含：
- **人格卡**（`30_人格熔爐.md`）— 完整的 AI 人格定義，含思維模式、專長、行為規範
- **角色卡**（`31_角色熔爐.md`）— 輕量版，專注於特定專案角色的職責和視角
- **分類彙整**（`output/01_分類彙整.md`）— 按領域分類的索引
- **原始素材**（`output/raw-cards/`）— 從各來源提煉的原始技能卡

## 核心分工

| 層 | 負責什麼 |
|----|----------|
| **專案管理（SRI）** | 結構、規則、流程、閘門、寫入邊界 |
| **熔爐百相（Furnace）** | 人格卡、角色卡、技能素材供應 |
| **MCP** | 外部工具連線（瀏覽器、API、第三方服務），設定在 `專案管理/_系統/基地/mcp/` |
