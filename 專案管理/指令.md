# 指令

> 第二態入口：你說一句話，AI 幫你做。
> 第一態：`_系統/基地/專案管理使用手冊.md`（自己操作）
> 第三態：`大絕招.md`（AI 自主推進）

---

## 你想做什麼？

### 🟢 開始一個新東西

> 「我想做一個 ___」

| 你說 | AI 做什麼 |
|------|-----------|
| `開一般專案 記帳App` | 建專案資料夾，幫你規劃流程 |
| `開程式專案 記帳App` | 同上，多開程式區和技術文件 |

還沒想好名字？直接說「我有個想法想聊聊」，AI 會先幫你釐清再開。

---

### 🔵 推進手上的專案

> 「接著做」「這個專案現在怎樣了」

| 你說 | AI 做什麼 |
|------|-----------|
| `討論專案` | AI 判斷你到哪了，接著推 |
| `看專案現況` | 一頁摘要：進度、缺口、下一步 |
| `顯示本輪成果` | 列出目前產出的文件 |
| `進入施工判定` | 檢查規劃是否夠完整可以動工 |

不想記指令？這些也行：
- 「這個專案現在到哪了」→ 看專案現況
- 「我要開始施工」→ 進入施工判定

---

### 🟡 想多幾個方向 / 收斂結論

> 「幫我多想幾個方向」「收斂一下」

| 你說 | AI 做什麼 |
|------|-----------|
| `發散思考 定價策略` | 多角色腦力激盪，先不下結論 |
| `收斂思考` | 把前一輪討論整理成可行方案 |
| `角色討論 要不要外包` | 多角色辯論，回你結論和分歧 |

---

### 🟠 整理混亂的東西

> 「好亂幫我整理」

| 你說 | AI 做什麼 |
|------|-----------|
| `整理 參考資料` | 掃描 + 分類 + 重新整理 |
| `手動對話整理` | 把這輪對話歸檔到系統紀錄 |
| `手動整理到總方案` | 把已確認結論寫進正式文件 |

自然說法也行：
- 「這邊好亂」→ 整理當前資料夾
- 「參考資料太亂了」→ 整理參考資料
- 「把已確認內容整理進總方案」→ 手動整理到總方案

---

### 🔴 讓 AI 自己跑

> 想讓 AI 自主推進整個專案？去看 **[大絕招.md](大絕招.md)**。

簡單版：說 `啟動 Harness`，AI 會先列計畫給你確認，然後自己跑。遇到重大決定會停下來問你。

## 完整指令參考

> 以下是所有指令的技術細節，日常用上面的就夠。

### 日常指令

| 指令 | 用途 |
|------|------|
| `開一般專案 <名稱>` | 建立一般模式專案 |
| `開程式專案 <名稱>` | 建立程式設計模式專案 |
| `討論專案` | AI 自動判斷階段並接續推進 |
| `看專案現況` | 摘要目前狀態、缺口與下一步 |
| `整理 <資料夾>` | AI 掃描 + 分類 + 整理指定資料夾 |
| `角色討論 <主題>` | 觸發多角色討論，回你結論與分歧 |
| `發散思考 <主題>` | 多角色發散，不急著下結論 |
| `收斂思考` | 把前一輪討論收成可採取方案 |
| `進入施工判定` | 檢查是否足以進入施工 |
| `顯示本輪成果` | 列出 `03_文件產出/` 最新成品 |
| `啟動 Harness` | 進入第三態自主推進 |

### 進階指令

| 指令 | 用途 |
|------|------|
| `手動對話整理` | 強制立即整理本輪對話 |
| `手動整理到總方案` | 強制把已確認內容融合進 `總方案.md` |
| `回流沙盒到主專案` | 把沙盒結論回主線，仍需你確認 |
| `比較所有沙盒` | 比較各探索或子專案的結論與風險 |
| `取用人格` | 從 Furnace 補人格卡到 `01_團隊/` |
| `列出團隊` | 列出當前容器的人格與角色 |
| `列出 MCP 伺服器` | 查看可用的 MCP server |
| `啟用 MCP <名稱>` | 啟用指定 MCP server |
| `停用 MCP <名稱>` | 停用指定 MCP server |
| `新增沙盒 <名稱>` | 建立探索或子專案 |
| `進入沙盒 <名稱>` | 切換到指定沙盒 |
| `升級沙盒` / `降級沙盒` | 探索與子專案之間轉換 |

### AI 背景自動做的事

- 對話內容分流歸檔到 `_系統/10-13`
- 偵測重大結論，同步到 `總方案.md`
- 收到素材自動整理到 `02_參考資料/`
- 偵測核心問題補進待確認清單
- 需要時主動建議角色討論

### 寫入邊界

| 區域 | 定位 | 說明 |
|------|------|------|
| `_系統/10-13` | 工作記錄 | 不是正式真值 |
| `總方案.md` | 規劃階段正式真值 | 已確認結論的彙整 |
| `02_參考資料/` | 素材層 | 不等於規格 |
| `03_文件產出/` | 成品層 | 不等於決策真值 |
| `04_主專案程式/` | 程式碼 | 改非 `.md` 前 AI 必停確認 |
| 沙盒 | 探索空間 | 結論不能直接跳進主線 |

### 路徑心智

- 日常看：`總方案.md`、`03_文件產出/`
- 偶爾翻：`01_團隊/`、`02_參考資料/`
- 程式模式才常看：`04_主專案程式/`
- `_系統/` 是 AI 工作台，通常不用手動開
- 舊資料統一放 `_系統/99_舊檔案封存區/`
