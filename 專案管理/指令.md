# 指令

> 第二態入口：指令驅動半自動。
> 第一態：`_系統/基地/專案管理使用手冊.md`
> 第三態：`大絕招.md`

## 日常高頻指令

| 指令 | 用途 |
|------|------|
| `開一般專案 <名稱>` | 建立一般模式專案 |
| `開程式專案 <名稱>` | 建立程式設計模式專案 |
| `討論專案` | AI 自動判斷階段並接續推進 |
| `看專案現況` | 摘要目前狀態、缺口與下一步 |
| `角色討論 <主題>` | 觸發多角色討論，回你結論與分歧 |
| `發散思考 <主題>` | 多角色發散，不急著下結論 |
| `收斂思考` | 把前一輪討論收成可採取方案 |
| `進入施工判定` | 檢查是否足以進入施工 |
| `顯示本輪成果` | 列出 `03_文件產出/` 最新成品 |
| `啟動 Harness` | 進入第三態自主推進 |

## AI 背景會自動做

- 把對話分流進 `_系統/10_對話整理.md` 到 `_系統/13_已確認結論.md`
- 偵測重大確認結論，視情況同步到 `總方案.md`
- 收到素材後整理到 `02_參考資料/`
- 偵測核心問題後補進 `_系統/12_待確認問題.md`
- 在需要時主動建議角色討論

## 進階指令

| 指令 | 用途 |
|------|------|
| `手動對話整理` | 強制立即整理本輪對話 |
| `手動整理到總方案` | 強制把已確認內容融合進 `總方案.md` |
| `回流沙盒到主專案` | 把沙盒結論回主線，仍需你確認 |
| `比較所有沙盒` | 比較各探索或子專案的結論與風險 |
| `取用人格` | 從 Furnace 補人格卡到 `01_團隊/` |
| `列出團隊` | 列出當前容器的人格與角色 |
| `列出 MCP 伺服器` | 查看 `_系統/基地/mcp/servers.json` 可用項 |
| `啟用 MCP <名稱>` | 啟用指定 MCP server |
| `停用 MCP <名稱>` | 停用指定 MCP server |
| `新增沙盒 <名稱>` | 建立探索或子專案 |
| `進入沙盒 <名稱>` | 切換到指定沙盒 |
| `升級沙盒` / `降級沙盒` | 探索與子專案之間轉換 |

## 路徑心智

- 你日常主要看：`總方案.md`、`03_文件產出/`
- 偶爾會翻：`01_團隊/`、`02_參考資料/`
- 程式設計模式才會常看：`04_主專案程式/`
- `_系統/` 是 AI 工作台，通常不用手動打開
- 舊資料封存統一放在 `_系統/99_舊檔案封存區/`

## 寫入邊界

- `_系統/10-13` 是工作記錄，不是正式真值
- `總方案.md` 是規劃階段正式真值
- `02_參考資料/` 是素材層，不等於規格
- `03_文件產出/` 是成品層，不等於決策真值
- 沙盒結論不能直接跳進主線正式文件
- 修改 `04_主專案程式/` 內任何非 `.md` 檔案前，AI 必停確認

## 自然說法也可以

- 幫我比較所有沙盒
- 我要開始施工
