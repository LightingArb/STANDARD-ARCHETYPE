# 天氣套利資料來源調查

- 建立日期：2026-04-02
- 研究主題：Polymarket 天氣市場需要的歷史預報快照與最終結算真值
- 研究狀態：第一輪盤點完成，尚未進入資料抓取與程式化驗證

## 目標市場

- 目標案例：
  - https://polymarket.com/zh-hant/event/highest-temperature-in-london-on-april-2-2026
- 類型：
  - 某日最高溫落在哪個整數攝氏區間的市場
- 站點：
  - London City Airport Station
  - 站代碼：`EGLC`

## Polymarket 規則重點

- 結算看的是 `EGLC` 在指定日期的最高溫。
- 結算來源是 Weather Underground 的日歷史頁：
  - https://www.wunderground.com/history/daily/gb/london/EGLC
- 結算單位是整數攝氏。
- 市場要等該日期資料 finalized 後才可完成結算。
- finalized 後若外部資料又被修正，規則寫明不再納入本市場結算。

## 使用者這輪真正需求

- 不是只要歷史實測溫度。
- 是要找「歷史上某個發布時刻，對未來某一天最高溫的預測值」。
- 之後要把這個預報值與最終結算真值做誤差統計，支援 YES / NO 判斷。

## 已找到的資料來源

### 1. Polymarket 市場規則頁

- 用途：
  - 確認市場類型、bucket、規則文字、結算站點與結算來源。
- 連結：
  - https://polymarket.com/zh-hant/event/highest-temperature-in-london-on-april-2-2026
- 初步判斷：
  - 是規則主來源，但不是歷史預報資料來源。

### 2. Weather Underground 日歷史頁

- 用途：
  - 取得 `EGLC` 指定日期的最終日最高溫。
- 連結：
  - https://www.wunderground.com/history/daily/gb/london/EGLC
- 初步判斷：
  - 是最接近市場最終真值的公開來源。
  - 適合當結算資料，不適合單獨解決歷史預報快照問題。

### 3. Weather Underground About Data

- 用途：
  - 了解 Wunderground 預報與觀測資料的公開說明。
- 連結：
  - https://www.wunderground.com/about/data
- 已知資訊：
  - 預報是其自家 forecast 產品，公開頁面說明 forecast 會小時更新。
- 目前限制：
  - 以目前公開文件與頁面來看，尚未找到可回放舊版 forecast 的公開 archive。
- 備註：
  - 「沒有公開歷史 forecast archive」這點目前是根據公開頁面做的推論，不是官方明文否定。

### 4. Met Office DataPoint API

- 用途：
  - 官方英國氣象資料來源，可取得 site list、forecast 與 observations。
- 連結：
  - https://www.metoffice.gov.uk/binaries/content/assets/metofficegovuk/pdf/data/datapoint_api_reference.pdf
- 已知資訊：
  - site-specific forecast 可取未來約 5 天。
  - forecast 會更新。
  - observations 可取近端觀測。
- 初步判斷：
  - 適合從現在開始存快照。
  - 不足以回補很久以前的歷史預報快照。

### 5. Met Office Weather DataHub FAQ

- 用途：
  - 查官方新資料平台保留範圍與更新頻率。
- 連結：
  - https://datahub.metoffice.gov.uk/support/faqs
- 已知資訊：
  - Land observations 回傳最近 48 小時。
  - atmospheric model files 保留最近 2 天。
- 初步判斷：
  - 適合 forward collection，不適合長期 historical backfill。

### 6. Met Office 歷史 forecast archive 說明

- 用途：
  - 確認是否存在容易取得的官方歷史預報 archive。
- 連結：
  - https://www.metoffice.gov.uk/research/library-and-archive/archive/forecast-data
- 初步判斷：
  - 官方 archive 存在，但不是這題要的「站點級、可直接用於 issue-time 回測」形式。

### 7. Open-Meteo Previous Runs API

- 用途：
  - 直接查「前一次、前幾次」forecast run 對未來的預測。
- 連結：
  - https://open-meteo.com/en/docs/previous-runs-api
- 初步判斷：
  - 是目前最接近需求、最容易上手的歷史預報快照來源。
  - 雖然不是 Wunderground 原版 forecast，但很適合先建立研究模型。

### 8. Open-Meteo Historical Forecast API

- 用途：
  - 查 archived forecasts，做 forecast vs observed accuracy 分析。
- 連結：
  - https://open-meteo.com/en/docs/historical-forecast-api
- 初步判斷：
  - 與 Previous Runs 可互補，適合快速驗證 lead time 誤差。

### 9. NOAA Open Data / GFS archive

- 用途：
  - 拿官方模式 archive，從 run time 回溯 forecast。
- 連結：
  - https://www.ncei.noaa.gov/products/ncei-data-noaa-open-dissemination-program
- 已知資訊：
  - 官方頁面列出 GFS、GEFS 等 forecast model 的 cloud access。
- 初步判斷：
  - 很適合做嚴格回測。
  - 但資料是 model/grid 層級，要自己處理格點、站點映射與 daily max 計算。

### 10. ECMWF historical archive

- 用途：
  - 取得高品質歷史 operational forecast archive。
- 連結：
  - https://www.ecmwf.int/en/forecasts/accessing-forecasts/order-historical-datasets
- 初步判斷：
  - 理論上很強，但完整 archive 取得門檻與成本較高。
  - 不適合作為第一步的快速驗證來源。

## 目前最重要的初步結論

- `最終真值` 最清楚的來源是：
  - Polymarket 規則頁 + Wunderground `EGLC` 日歷史頁
- `歷史預報快照` 最可行的第一步來源是：
  - Open-Meteo Previous Runs API
  - Open-Meteo Historical Forecast API
- `只用官方 forecast archive` 的備選路線是：
  - NOAA GFS archive
  - ECMWF archive
- `Met Office` 更適合從現在開始自行存快照，不適合長期回補。
