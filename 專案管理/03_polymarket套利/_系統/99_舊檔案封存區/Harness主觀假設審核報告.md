## 正確的工作順序

```
Phase 0：數據收集（不做任何參數決策）
├── 收集歷史預報 vs 實際觀測
├── 收集 Polymarket 歷史價格、spread、結算
├── 確認平台參數（最低下注額、手續費）

Phase 1：數據分析（用數據填入所有空白參數）
├── 計算各模型實際 sigma（取代假設的 0.3°C）
├── 計算模型間不一致率和相關性
├── Forward selection 決定模型數量
├── 交叉驗證決定 alpha
├── 從盈利條件反推所有門檻值

Phase 2：Paper Trading（用真實模擬填入剩餘參數）
├── 估計勝率 → 算連虧暫停閾值
├── 收集 daily P&L 分佈 → 算日虧損暫停線
├── 估計 edge variance → 校正 Kelly fraction
├── 持續到統計精度達標（不是固定天數）

Phase 3：Live Trading（所有參數此時都有數據支持）
```

## 核心原則

**方法論是結論，數字不是。**

「需要風控」是結論。「10%」不是。
「需要 ensemble」是結論。「3 個模型」不是。
「需要 paper trading」是結論。「30 天」不是。

所有具體數值都是待驗證的空白欄位，不是答案。
