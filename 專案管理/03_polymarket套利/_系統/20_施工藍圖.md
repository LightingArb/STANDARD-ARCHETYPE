# 施工藍圖

- 狀態：草稿
- 最後更新：2026-04-02
- 版本：v0.2

## 第一階段模組邊界

- 真值層：
  - 負責讀取 Polymarket 規則與 Weather Underground `EGLC` 日歷史頁的最終結算值。
- 預報快照層：
  - 負責取得 issue-time forecast snapshot，來源候選包含 Open-Meteo、NOAA、ECMWF、Met Office。
- 分析層：
  - 負責計算 `forecast_error`、`remaining_rise` 與 lead time 分布。
- 訊號層：
  - 負責把 YES / NO 兩條統計線轉成可解釋的決策門檻。

## 初版 API / 資料契約草案

- `weather_market_snapshot`
  欄位至少包含 `market_id`、`event_date_local`、`bucket_prices`、`timestamp`。
- `forecast_snapshot`
  欄位至少包含 `forecast_source`、`station_id`、`snapshot_time_local`、`snapshot_time_utc`、`forecast_daily_high`。
- `weather_truth`
  欄位至少包含 `station_id`、`event_date_local`、`final_daily_high`、`finalized_at`。
- `weather_signal_feature`
  欄位至少包含 `lead_hours_to_end_of_day`、`forecast_error`、`remaining_rise`、`max_temp_so_far`。
