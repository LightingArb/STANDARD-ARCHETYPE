# 隨機漫步模型應用在溫度預測的方法論研究

> 研究日期: 2026-04-01
> 目的: 將外匯隨機漫步框架移植到 Polymarket 天氣市場溫度預測

---

## 一、溫度 vs 金融市場：隨機過程的根本差異

| 特性 | 外匯價格 | 溫度 |
|------|---------|------|
| 趨勢 | 近似隨機漫步 (unit root) | **無 unit root**，有季節性趨勢 |
| 均值回歸 | 弱或無 | **強均值回歸** -- 偏離季節均值會被拉回 |
| 週期性 | 無內在週期 | **日週期** (白天升夜晚降) + **年週期** (季節) |
| 波動率 | 隨機、可能叢聚 (GARCH) | 有季節性模式，夏天波動率與冬天不同 |
| 分布 | 接近對數常態，有肥尾 | 接近常態，但有季節性偏移 |
| 跳躍 | 事件驅動 (央行決議等) | 天氣鋒面、系統性天氣事件 |

**核心結論: 溫度不是隨機漫步。** 直接套用 GBM (Geometric Brownian Motion) 會嚴重錯誤。你需要的是 **帶有確定性成分 (日週期 + 季節) 的均值回歸隨機過程**。

## 二、學術界 & 產業界的溫度隨機模型

### 2.1 標準模型: 帶季節性的 Ornstein-Uhlenbeck (OU) 過程

這是天氣衍生品定價的「工業標準」。Benth et al. (2011) 的經典框架:

```
dT(t) = [dS(t)/dt + kappa * (S(t) - T(t))] dt + sigma(t) dW(t)
```

其中:
- `T(t)`: 時間 t 的溫度
- `S(t)`: 季節性均值函數 (用截斷 Fourier 級數擬合)
- `kappa`: 均值回歸速度 (溫度拉回季節均值的力道)
- `sigma(t)`: 季節性波動率 (也用 Fourier 級數)
- `W(t)`: 標準布朗運動

**直覺類比:**
- 就像外匯中的 Vasicek 利率模型
- `kappa` 對應你在外匯中的「均值回歸強度」
- `S(t)` 就是溫度的「公允價值」-- 隨季節和日內時間變化

### 2.2 進階變體

| 模型 | 改進點 | 適用場景 |
|------|--------|---------|
| OU + GARCH 波動率 | 波動率叢聚 | 波動率本身不穩定時 |
| OU + Levy 過程 | 容許跳躍 (temperature spikes) | 極端天氣事件 |
| 分數 OU (Fractional OU) | 長記憶效應 | 多日預測 |
| Regime-switching OU | 不同天氣狀態切換 | 鋒面過境等結構性變化 |
| 時變均值回歸速度 | kappa 隨時間變化 | CME 天氣期貨定價 (Kent 大學研究) |

### 2.3 CME 天氣期貨市場

CME 交易的天氣衍生品主要基於溫度指數:
- **HDD** (Heating Degree Days): max(65F - T, 0)
- **CDD** (Cooling Degree Days): max(T - 65F, 0)
- **CAT** (Cumulative Average Temperature)

定價模型普遍使用帶季節性的 OU 過程，用歷史資料校準參數後，透過 Monte Carlo 模擬或解析公式定價。

## 三、日內溫度路徑建模

這是你的 Polymarket 場景最核心的部分: **從中午 12 點 14C 出發，預測到午夜的溫度路徑**。

### 3.1 確定性成分: 日內溫度曲線

日內溫度有很強的確定性模式，可用分段正弦函數近似:

```
日出 ~ 下午2-3點:  上升段 (正弦函數)
下午2-3點 ~ 日落:  緩降段 (正弦函數)  
日落 ~ 次日日出:    夜間降溫段 (指數衰減 / 平方根函數)
```

經典的 **sine-exponential 模型** 在晴天表現最佳，陰天精度下降。

**關鍵參數:**
- T_max: 當日預測最高溫 (通常 14:00-15:00)
- T_min: 當日預測最低溫 (通常 05:00-06:00)
- 日出/日落時間

### 3.2 隨機成分: 日內 OU 過程

在確定性日內曲線上，疊加一個均值回歸隨機擾動:

```
T(t) = S_diurnal(t) + X(t)

dX(t) = -kappa_intraday * X(t) dt + sigma_intraday dW(t)
```

其中 `S_diurnal(t)` 是日內確定性曲線，`X(t)` 是圍繞它的隨機偏差。

### 3.3 實際操作: 你的場景

**已知:** 中午 12:00，溫度 14C (倫敦，假設 4 月)

**步驟:**
1. 查詢當日天氣預報取得 T_max, T_min 預測值
2. 建立日內確定性曲線 S(t)
3. 計算目前偏差: X(12:00) = 14 - S(12:00)
4. 從 X(12:00) 出發，模擬 OU 過程到午夜
5. Monte Carlo 跑 10,000 條路徑
6. 統計每條路徑的 max(T) 落在哪個整數區間
7. 得到各 bucket 的機率分布

## 四、溫度的「震幅」-- 日內 High-Low Range

### 4.1 Diurnal Temperature Range (DTR)

氣象學中，**DTR = T_max - T_min** 是一個被廣泛研究的獨立變數。

與金融市場 realized volatility 的類比:

| 金融 | 溫度 |
|------|------|
| Daily High-Low Range | DTR (日溫差) |
| Realized Volatility (intraday) | 日內溫度標準差 |
| Parkinson estimator | 可以直接套用到 DTR |
| VIX (implied vol) | 天氣預報的不確定性區間 |

### 4.2 DTR 作為獨立信號

- DTR 有明確的季節性 (春秋大、夏冬小)
- DTR 受雲量、風速、濕度影響
- 陰天 DTR 小 (雲層抑制白天升溫 + 夜間降溫)
- 晴天 DTR 大

**實用啟示:** 如果你能預測當天的 DTR，就能大幅縮小溫度分布的不確定性。DTR 本身可以用前一天的天氣條件 + 當天雲量預報來估計。

### 4.3 DTR 的隨機建模

Whan & Zwiers (2018) 的研究提出了對平均日溫和 DTR 的聯合隨機分析框架:
- 兩者都用截斷 Fourier 級數去季節化
- 殘差用 FARIMA 模型 (捕捉長記憶效應)
- 兩者之間存在統計相依性

## 五、實際可落地的方法

### 5.1 最簡單可行方案 (MVP): Ensemble Forecast 直接用

**不需要自己建模。** 目前 Polymarket 天氣 bot 的主流做法:

1. **資料源:** Open-Meteo API (免費、無需認證)
2. **核心:** GFS 31-member ensemble forecast
3. **方法:** 抓 31 條 ensemble member 的溫度預報，數有幾條落在各 bucket
4. **機率:** 如 28/31 members 預測 > 20C，則 P(>20C) = 90.3%
5. **下注:** 比較機率 vs Polymarket 報價，正 EV 就下

**優點:** 簡單、快速、利用了專業氣象模型的全部能力
**缺點:** 只有 31 個 samples，機率估計粒度粗

### 5.2 進階方案: Ensemble + 隨機模型混合

1. 用 ensemble mean 作為確定性預報
2. 用 ensemble spread 校準 sigma
3. 在 ensemble 基礎上跑 OU Monte Carlo，生成更多路徑
4. 得到更平滑的機率分布

### 5.3 完整方案: 多模型加權

目前表現最好的 Polymarket 天氣 bot 使用 4-model ensemble:

| 模型 | 權重 |
|------|------|
| ECMWF | 35% |
| GFS | 25% |
| UKMO | 20% |
| NWS | 20% |

配合 Bayesian edge detection + Kelly criterion 做倉位管理。

### 5.4 Python 工具鏈

| 用途 | 工具 |
|------|------|
| 天氣資料 | `open-meteo` API (免費，RESTful) |
| SDE 模擬 | `sdeint`, `sdepy`, 或直接用 `numpy` 寫 Euler-Maruyama |
| OU 過程 | `scipy` + 手寫 (OU 過程的 Euler-Maruyama 只需 3 行) |
| 波動率建模 | `arch` 套件 (GARCH) |
| 時間序列 | `statsmodels` (ARIMA, Fourier 去季節化) |
| 資料處理 | `pandas`, `numpy` |

### 5.5 OU 過程 Euler-Maruyama 模擬核心程式碼 (概念)

```python
import numpy as np

def simulate_temperature_ou(T_current, T_target_curve, kappa, sigma, dt, n_steps, n_paths):
    """
    T_current: 當前溫度 (e.g., 14.0)
    T_target_curve: 長度 n_steps 的陣列，日內確定性曲線上的目標值
    kappa: 均值回歸速度
    sigma: 波動率
    dt: 時間步長 (e.g., 1/24 天 = 1小時)
    n_steps: 模擬步數
    n_paths: Monte Carlo 路徑數
    """
    paths = np.zeros((n_paths, n_steps + 1))
    paths[:, 0] = T_current

    for i in range(n_steps):
        dW = np.random.randn(n_paths) * np.sqrt(dt)
        drift = kappa * (T_target_curve[i] - paths[:, i])
        paths[:, i+1] = paths[:, i] + drift * dt + sigma * dW

    return paths

# 用法:
# paths = simulate_temperature_ou(14.0, target_curve, kappa=2.0, sigma=1.5, dt=1/24, n_steps=12, n_paths=10000)
# max_temps = paths.max(axis=1)
# 統計 max_temps 落在各 bucket 的比例
```

## 六、從外匯框架到溫度框架的對照表

| 你在外匯做的事 | 溫度場景的對應 |
|--------------|-------------|
| 用 GBM 模擬未來價格路徑 | 用 OU 過程模擬未來溫度路徑 |
| 隱含波動率 from options | Ensemble spread 或歷史 DTR |
| 計算到期時的價格分布 | 計算到結算時的溫度分布 |
| 各 strike 的機率 | 各 bucket 的機率 |
| 比較理論價 vs 市場價找 edge | 比較理論機率 vs Polymarket 報價找 edge |
| Kelly criterion 決定倉位 | 同樣適用 |
| Greeks 管理風險 | 溫度對時間的敏感度 (越接近結算不確定性越低) |
