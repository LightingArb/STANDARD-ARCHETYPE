## S-03. 指令.md（結構要求）

生成後的 `指令.md` 必須符合以下條件：

1. **第一行必須直接是 `## 常用指令`。**
2. 內容只講第二層，不能把第一層與第三層混成一份。
3. 專案建立命令一律使用：
   - `新增一般專案 名稱`
   - `新增程式專案 名稱`
4. 必須新增以下 v4 指令：
   - `角色討論 主題`
   - `發散思考 主題`
   - `收斂思考`
   - `列出 MCP 伺服器`
   - `啟用 MCP 名稱`
   - `停用 MCP 名稱`
5. 必須明講 10-15 自動更新仍在背景持續運作，`對話整理` 只是手動強制入口。
6. 必須明講團隊位置為 `01_團隊/`，並以同一個團隊總覽命令同時檢視人格與角色。
7. 必須明講程式碼修改閘門：
   - 改 `04_主專案程式/` 內任何非 `.md` 檔案前必停確認。
8. 必須把沙盒、執行單元、團隊、MCP、Harness 分章節說清楚。
9. 所有最終路徑一律用 v4 正式路徑，不使用 `-v4` 路徑。

## S-04. 大絕招.md（結構要求）

生成後的 `大絕招.md` 必須符合以下條件：

1. 開頭必須有 `## 啟動 Harness` 段落，說明：
   - `進入專案 <名稱>`
   - `啟動 Harness`
   - `自主推進到 [目標]`
   - `自主完成這個專案`
2. 必須先做 Step 0 目標解析與推進計畫，等使用者確認才開始。
3. 必須明講 Harness 的 `.md` 完全自主邊界與三個硬閘門：
   - 改 `04_主專案程式/` 內非 `.md` 檔案
   - 刪除 / 搬移 / 覆蓋大量檔案
   - MCP 首次連線或權限升級
4. 必須明講 Harness 內可以自主觸發角色討論，並寫入 `01_團隊/角色會議紀錄.md`。
5. 停止機制必須改為 v4：
   - AI 自主判斷何時停
   - 最多連續 10 輪自主推進，必須停一次報告
   - 使用者可隨時 `暫停` / `停止`
6. 不得保留舊版計數器式停止主機制。
7. 進度報告必須簡潔，重點是「做到哪、為什麼停、下一步」。

根 README 必須：

   - `專案管理/_templates/`
   - `專案管理/99_基地/`
   - `專案管理/99_基地/mcp/`

## S-08. 人格配發規格

> 本節不是模板檔案，而是描述專案 `01_團隊/` 的配發邏輯。
> 供完整初始化、新增專案、升級專案、系統更新時參照。

### 配發規則

| 時機 | 行為 |
|------|------|
| 新增一般專案 | 複製規劃角色 → 掃角色依賴 → 聯集去重 → 複製人格到 `01_團隊/` |
| 新增程式專案 | 複製完整角色 → 掃角色依賴 → 聯集去重 → 複製人格到 `01_團隊/` |
| 升級專案 | 補入新角色 → 掃新角色依賴 → 補入缺少人格 |
| 新增子專案 | 同程式專案 |
| 新增探索 | 不複製，繼承所屬正式容器 |
| 使用者說 `取用人格` | 走 Furnace 取用流程，補到 `01_團隊/` |
| 需要客製 | 走 Furnace L3 客製流程，產出存入 `01_團隊/` |

### 自動配發結果

| 模式 | 角色 | 人格（聯集去重） |
|------|------|----------------|
| 一般 | 01-04（4 張） | #101, #102, #103, #104, #211, #213（6 張） |
| 程式設計 | 01-04 + 11-13（7 張） | #101, #102, #103, #104, #202, #211, #213（7 張） |

### 檔名規則

- 人格：`人格_101_通用核心型.md`
- 角色：`角色_01_策略決策.md`
- 會議紀錄：`角色會議紀錄.md`

## S-09. MCP 三件組

### README.md（骨架版）

```markdown
# MCP 工具層

這裡放 SRI 的 MCP 設定與安全政策。

## 檔案
- `README.md`：本檔
- `servers.json`：可用 server 清單
- `policies.md`：安全政策

## 原則
1. 所有 MCP server 預設為 read-only
2. 首次連線必須使用者確認
3. 會寫入外部系統的 server 永遠需要使用者確認
4. MCP 產物依路由寫入 02 / 03 / 04
```

### servers.json（骨架版）

```json
{
  "servers": []
}
```

### policies.md（骨架版）

```markdown
# MCP 安全政策

1. 所有 MCP server 預設為 read-only。
2. 首次連線必須使用者確認。
3. 會寫入外部系統的 server 永遠需要使用者確認。
4. Harness 中的 MCP 呼叫遵守同樣的硬閘門。
5. 原始輸入寫入 `02_參考資料/raw_input/`。
6. AI 消化後的分析寫入 `03_文件產出/`。
7. 實際程式碼整合寫入 `04_主專案程式/`，且先通過程式碼修改閘門。
```
