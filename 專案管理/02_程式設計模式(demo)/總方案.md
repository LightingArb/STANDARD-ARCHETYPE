# 總方案

- 狀態：已確認可施工
- 專案模式：程式設計
- 專案主題：對沖交易系統
- 技術棧：Python、FastAPI、PostgreSQL、Redis、Next.js、Docker
- 最後更新：2026-04-01
- 版本：demo-v1

## 系統目標

- 建立一套可監控、可回測、可下單的對沖交易系統，先支援單一交易所 MVP。
- 第一階段主打現貨與永續合約之間的價差觀察、下單與風控。

## 核心功能

- 行情接收：訂閱交易所 WebSocket，維持報價與倉位快照。
- 策略引擎：根據價差、資金費率與滑價門檻決定是否進場。
- 風控層：限制單筆風險、總曝險與 API 失敗重試次數。
- 監控面板：顯示倉位、PnL、告警與服務健康度。

## 成功條件

- 回測報告能穩定輸出收益、回撤與成交品質指標。
- API 服務與前端面板可在本地 Docker 環境一次啟動。
- 任一風控條件觸發時，系統能在 3 秒內停止新單並發出告警。

## 目前施工策略

- 先拆成前端頁面、API 服務與整合層三個執行單元。
- 第一期只做幣安資料源，其他交易所放在資料比較與介面設計層。
- 支付模組獨立成子專案處理，避免核心交易流程與資金流混在一起。
