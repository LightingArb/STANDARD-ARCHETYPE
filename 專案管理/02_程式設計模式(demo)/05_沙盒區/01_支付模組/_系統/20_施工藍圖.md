# 施工藍圖

- 狀態：已確認可施工
- 最後更新：2026-04-01
- 版本：demo-v1

## 模組邊界

- Payment API：建立請求、查詢狀態、接收人工重試指令。
- State Machine：管理 pending、processing、succeeded、failed、manual_review 五個主要狀態。
- Reconciliation：每日比對支付紀錄與帳務結果，產生差異清單。

## API 契約 sample

- `POST /payments`：建立支付請求，回傳 `payment_request_id`。
- `GET /payments/{id}`：查詢當前狀態與事件歷史。
- `POST /payments/{id}/retry`：人工或自動觸發重試。
