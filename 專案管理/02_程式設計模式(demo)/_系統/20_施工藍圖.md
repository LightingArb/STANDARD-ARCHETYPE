# 施工藍圖

- 狀態：已確認可施工
- 最後更新：2026-04-01
- 版本：demo-v1

## 模組邊界

- 前端：只讀取 API 提供的倉位、PnL、告警與策略狀態，不直接觸碰交易所。
- API 服務：負責行情收集、帳戶查詢、下單委託與策略控制指令。
- 策略與風控：由後端服務內部模組處理，對外只暴露狀態與指令介面。

## API 契約 sample

- `GET /api/v1/positions`：回傳當前倉位、未實現損益與保證金使用率。
- `POST /api/v1/strategy/start`：啟動指定策略，需附策略代號與環境參數。
- `POST /api/v1/risk/kill-switch`：立即停止新單並切換系統為保護模式。

## 施工順序

1. 先完成行情與帳戶查詢接口。
2. 再補策略啟停與風控 kill switch。
3. 最後整合前端與壓力測試場景。
