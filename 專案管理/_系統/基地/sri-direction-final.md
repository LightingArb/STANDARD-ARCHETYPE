# SRI v4.1.0 當前方向總結

> 日期：2026-04-01
> 狀態：v4.1 封裝層遷移完成

## 一、v4.1 的核心改動

1. 使用者視角收斂成 `指令.md`、`大絕招.md`、專案資料夾。
2. 基地層封裝進 `專案管理/_系統/基地/`。
3. templates 封裝進 `專案管理/_系統/templates/`。
4. 每個專案把中間工作檔收進 `_系統/`。
5. `總方案.md` 成為專案根唯一正式方案檔。

## 二、系統全貌

```text
Furnace（人格供應層）    SRI（結構與規則層）           Harness（自治層）
  熔爐百相/               專案管理/_系統/基地/sa/         專案管理/大絕招.md
  人格卡 + 角色卡         6 份 sri-*.md source           閘門式自主推進
```

## 三、入口與系統層

| 檔案 | 角色 |
|------|------|
| `_系統/基地/專案管理使用手冊.md` | 第一態入口 |
| `指令.md` | 第二態入口 |
| `大絕招.md` | 第三態入口 |
| `_系統/基地/人格映射表.md` | 團隊與 Furnace 對接說明 |
| `_系統/基地/系統總覽.md` | 路由與狀態 |
| `_系統/基地/README.md` | 基地層說明 |
| `_系統/基地/mcp/*` | MCP 設定層 |
| `_系統/templates/` | demo 模板 |

## 四、v4.1 硬規則

1. `總方案.md` 是規劃階段正式真值。
2. 專案 `_系統/` 是 AI 工作台，使用者通常不用管。
3. 10-15 會背景自動更新，手動整理只是強制入口。
4. 探索結論不得直接跳進主線正式文件。
5. `04_主專案程式/` 內非 `.md` 修改前必停確認。
6. MCP 首次連線或權限升級必停確認。

## 五、操作心智

### 一般模式

```text
先收斂 總方案.md
  → 持續整理 _系統/15_方案變更紀錄.md
  → 產出文件到 03_文件產出/
```

### 程式設計模式

```text
總方案.md
  → _系統/20_施工藍圖.md
  → _系統/30_施工進度.md
  → 04_主專案程式/
```

### 探索回流

```text
沙盒 _系統/13_已確認結論.md
  → 主線 _系統/12_待確認問題.md
  → 主線 _系統/13_已確認結論.md
  → 主線 總方案.md
```
