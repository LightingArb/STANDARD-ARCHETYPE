## 施工步驟

### 步驟 1：備份現有團隊

將 `03_polymarket套利/01_團隊/` 下所有現有檔案複製一份到 `03_polymarket套利/_系統/99_舊檔案封存區/01_團隊_v4.0_backup/`。

如果 `99_舊檔案封存區/` 不存在，先建立。

---

### 步驟 2：確認該專案的模式與應配清單

讀取 `03_polymarket套利/總方案.md` 的 `專案模式` 欄位。

若為程式設計模式：
- 應配角色：7 張（01_策略決策、02_創意總監、03_使用者研究、04_技術聯絡、11_系統架構師、12_安全風控、13_品質驗證）
- 應配人格：7 張（#101, #102, #103, #104, #202, #211, #213）

若為一般模式：
- 應配角色：4 張（01-04）
- 應配人格：6 張（#101, #102, #103, #104, #211, #213）

---

### 步驟 3：從 Furnace 真實複製人格卡

對每張應配人格卡：

1. 從 `熔爐百相/鍛造間/人格/[編號]_[名稱].md` 讀取完整內容
2. 建立 `03_polymarket套利/01_團隊/人格_[編號]_[名稱].md`
3. 檔案結構如下：

```markdown
# 人格_[編號]_[名稱]

<!-- furnace_source: 熔爐百相/鍛造間/人格/[編號]_[名稱].md -->
<!-- furnace_version: [原檔的編譯日期] -->
<!-- last_synced: [今天日期] -->

## ===== 基底區（來自 Furnace，系統更新時可刷新）=====

[完整貼入 Furnace 原始內容]

## ===== 專案累積區（本專案獨有，系統更新不可動）=====

（尚無專案累積記憶）
```

4. ⚠️ 不得修改 Furnace 原始內容，完整複製
5. ⚠️ 不得使用 placeholder 或摘要版

---

### 步驟 4：從 Furnace 真實複製角色卡

對每張應配角色卡：

1. 從 `熔爐百相/鍛造間/角色/[編號]_[名稱].md` 讀取完整內容
2. 建立 `03_polymarket套利/01_團隊/角色_[編號]_[名稱].md`
3. 檔案結構同人格卡（基底區 + 專案累積區）
4. metadata 的 furnace_source 指向角色路徑

---

### 步驟 5：確認角色會議紀錄

確認 `03_polymarket套利/01_團隊/角色會議紀錄.md` 存在。
若已存在且有內容，不動。
若不存在或為空，從 sri-templates.md 的 T-T0 模板生成。

---

### 步驟 6：跑 §2-1A 完整自檢

對 `03_polymarket套利/01_團隊/` 執行五層自檢：

```
A. 數量檢查
□ 人格卡數量 = ?（應配 vs 實際）
□ 角色卡數量 = ?（應配 vs 實際）
□ 角色會議紀錄.md 存在？

B. 空殼防呆檢查
□ 每張人格卡 bytes = ?（需 > 1000）
□ 每張角色卡 bytes = ?（需 > 500）
□ 是否包含 "Template placeholder"？
□ 是否包含 "應從 Furnace 配發"？
□ 角色卡是否有空白 "角色定位："？

C. 必備章節檢查
□ 人格卡有 metadata / 基底區 / 專案累積區？
□ 角色卡有 角色定位 / 職責 / 預設人格 / 本輪最終建議？

D. 來源型別檢查
□ 不含 template_only？
□ 不含 demo_only？

E. 合格判定
□ 全部通過 → Phase 2 成功
□ 任一失敗 → 報告失敗項，重新修復
```
